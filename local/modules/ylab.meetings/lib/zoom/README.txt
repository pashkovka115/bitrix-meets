
0. Внимание!!!
Перед подключением этого компонента убедитесь в работоспособности вашего кода.
В противном случае сервис Zoom может забанить за большое количество обращений к нему.

1. Создать URL для редиректа Zoom.
Например в корне сайта создать папку двух-уровневую http://site.com/oauth-zoom-ylab/authorize и подключить компонент meeting.oauth_redirect
К первому уровню доступ запрещён со статусом 403, но если ЗНАТЬ абсолютный адрес то к нему можно спокойно обращаться.

2. Дать разрешение на чтение этой директории (oauth-zoom-ylab/authorize) из админки Битрикса
3. В аккаунте Zoom создать приложение "OAuth".
   Выбрать scope: "meeting:write:admin", можно добавить нужные.
   Для тестирования модуля отправлять приложение на модерацию не надо.
4. Созданный URL внести в приложение Zoom в "Redirect URL for OAuth" и "Add allow lists"
   и в настройки модуля "URL редиректа"
5. Так же должна существовать ссылка для начала авторизации вида:
   https://zoom.us/oauth/authorize?response_type=code&client_id=" . CLIENT_ID . "&redirect_uri=" . REDIRECT_URI
   (Вдальнейшем ссылка будет интегрирована в модуль и дополнительное действие не понадобится)

PS: Стандарт авторизации "Oauth 2.0" работает только на сервере. Чтобы он заработал на локальной машине необходимо
    установить прокси. Например https://ngrok.com/ или аналогичный. И настроить локальную маршрутизацию.
    Если OpenServer:
    сделать алиас localhost -> ваш-сайт-для-разработки.loc
    запустить: ngrok http 80
    сделать алиас: адрес-из-консоли.ngrok.io -> ваш-сайт-для-разработки.loc
    Ваш локальный сайт доступен из интернета по адресу: адрес-из-консоли.ngrok.io!
